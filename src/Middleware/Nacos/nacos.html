Nacos ​

Nacos 简介 ​

Nacos 是阿里巴巴开源的一个动态服务发现、配置管理和服务管理平台。它提供了服务注册、发现、配置管理等功能，广泛应用于阿里巴巴、百度等公司。

Nacos 特性 ​

  • 服务注册与发现
  • 配置中心
  • 服务管理
  • 服务健康检查
  • 服务降级
  • 服务熔断
  • 服务限流

Nacos 架构 ​

Nacos 架构主要由以下几个部分组成：

  • 服务提供者：提供服务的一方，需要将自己的服务注册到 Nacos 中。
  • 服务消费者：使用服务的一方，需要从 Nacos 中获取服务提供者的信息，并调用服务。
  • 服务注册中心：负责服务注册与发现的组件，提供服务注册、发现、健康检查等功能。
  • 配置中心：负责配置管理的组件，提供配置管理、动态刷新等功能。
  • 服务网关：负责服务路由、负载均衡等功能。
  • 服务治理：负责服务治理的组件，提供服务治理、限流、熔断等功能。

Nacos 核心类 ​

  • NacosDiscoveryProperties：Nacos 配置类，负责配置 Nacos 的配置。
  • NacosDiscoveryClient：Nacos 客户端类，负责服务注册与发现的组件。
  • NacosDiscoveryProperties：Nacos 配置类，负责配置 Nacos 的配置。
  • NacosServiceRegistry：Nacos 服务注册类，负责服务注册的组件。

Nacos + Spring Cloud Gateway + Spring Boot 实战：实现路由、鉴权、限流、降级、黑白名单 ​

3. 服务提供者 ​

在application.yml配置文件中添加Nacos和Gateway的相关配置，具体配置如下：

yaml
spring:
  application:
    name: service-provider
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848  # Nacos 服务地址
        namespace: public           # 命名空间，默认public
        group: DEFAULT_GROUP        # 分组，默认DEFAULT_GROUP
        cluster-name: DEFAULT       # 集群名称，默认DEFAULT
        weight: 1                   # 权重，默认1
        enabled: true               # 是否开启服务注册
        register-enabled: true      # 是否注册服务
        metadata:                   # 元数据
          version: 1.0.0
          env: dev

server:
  port: 8081                       # 服务端口
management:
  endpoints:
    web:
      exposure:
        include: '*'

6. 服务治理 ​

在application.yml配置文件中添加Nacos和Gateway的相关配置，具体配置如下：

yaml
spring:
  cloud:
    nacos:
      discovery:
        health-check-enabled: true  # 开启健康检查
        health-check-interval: 10s  # 健康检查间隔
        health-check-timeout: 5s    # 健康检查超时时间
        health-check-fail-threshold: 3  # 健康检查失败阈值
        health-check-pass-threshold: 1  # 健康检查通过阈值
        heartbeat-interval: 5s      # 心跳间隔
        heartbeat-timeout: 15s      # 心跳超时时间
        fail-fast: true             # 快速失败
        watch:
          enabled: true             # 开启服务监听
          delay: 1s                 # 监听延迟
          timeout: 10s              # 监听超时时间
        service:
          name: service-consumer    # 服务名称
          port: 8082                # 服务端口
          ip: 127.0.0.1             # 服务IP
          enabled: true             # 是否启用服务
          register-enabled: true    # 是否注册服务
          discovery-enabled: true   # 是否开启服务发现

9. 鉴权 ​

在application.yml配置文件中添加Nacos和Gateway的相关配置，具体配置如下：

yaml
spring:
  cloud:
    nacos:
      discovery:
        service:
          name: service-consumer    # 服务名称
          port: 8082                # 服务端口
          ip: 127.0.0.1             # 服务IP
          enabled: true             # 是否启用服务
          register-enabled: true    # 是否注册服务
          discovery-enabled: true   # 是否开启服务发现
          auth:
            enabled: true            # 是否开启鉴权
            username: admin         # 鉴权用户名
            password: admin123      # 鉴权密码
            token: abcdefg123456    # 鉴权token
            secret-key: mysecretkey # 密钥
            access-key: myaccesskey # 访问密钥
            role: admin            # 角色
            permission: read       # 权限
            policy: read          # 策略
            scope: read           # 范围
            resource: read        # 资源
            action: read         # 操作
            principal: admin     # 主体
            credential: admin123 # 凭证
            signature: abcdefg123456 # 签名
            timestamp: 1234567890 # 时间戳
            nonce: abcdefg123456 # 随机数
            version: 1.0.0 # 版本
            algorithm: hmac-sha256 # 算法
            header: Authorization # 头
            prefix: Bearer # 前缀

10. 限流 ​

在application.yml配置文件中添加Nacos和Gateway的相关配置，具体配置如下：

yaml
spring:
  cloud:
    gateway:
      routes:
        - id: rate_limiter_route
          uri: http://example.org
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 10  # 每秒允许的请求数
                redis-rate-limiter.burstCapacity: 20  # 最大突发请求数
                redis-rate-limiter.requestedTokens: 1  # 每个请求消耗的令牌数
                key-resolver: "#{@userKeyResolver}"  # 限流key解析器
  redis:
    host: localhost
    port: 6379
    password: 
    database: 0

          uri: http://example.org
          filters:
            - name: RequestRateLimiter
              args:
                key-resolver: "#{@userKeyResolver}"  # 限流key解析器

@Bean
public KeyResolver userKeyResolver() {
    return exchange -> Mono.just(exchange.getRequest().getRemoteAddress().getHostName());
}
