Git Base(git 基础知识和开发使用) ​

Git 基础知识 ​

Git 简介 ​

Git 是目前最流行的版本控制系统，被誉为史上最优秀的版本控制系统。Git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 与其它版本控制系统相比有以下几点不同：

  1. Git 是一个分布式版本控制系统，这意味着它可以克隆一个仓库到本地，然后在本地对仓库进行修改，而无需联网。
  2. Git 是一个可靠的版本控制系统，它保证了完整性，确保了每一次提交的版本都是可用的。
  3. Git 是一个开源的版本控制系统，这意味着任何人都可以免费使用 Git，并帮助开发者进行协作。
  4. Git 是一个功能强大的版本控制系统，它支持多种工作流，包括集中式工作流、功能分支工作流、发布工作流等。
  5. Git 是一个高效的版本控制系统，它在本地操作时速度快，在网络上进行版本控制时速度也快。

Git 工作流 ​

Git 工作流是 Git 最重要的概念之一。Git 工作流是指在日常开发中，使用 Git 进行代码版本控制的具体流程。Git 工作流有很多种，常用的有以下几种：

  1. 集中式工作流：集中式工作流就是将代码库存放在中央服务器，开发者直接从中央服务器克隆代码到本地，然后在本地进行开发，开发完成后再将代码推送到中央服务器。集中式工作流是最简单的工作流，但是它不支持多人同时开发，也不支持分支开发。
  2. 功能分支工作流：功能分支工作流是将代码库存放在中央服务器，开发者克隆代码到本地，然后在本地创建分支进行开发，开发完成后再将分支合并到主分支。功能分支工作流支持多人同时开发，也支持分支开发。
  3. 发布工作流：发布工作流是将代码库存放在中央服务器，开发者克隆代码到本地，然后在本地进行开发，开发完成后将代码打包，然后发布到测试环境进行测试，测试通过后再发布到生产环境。发布工作流支持多人同时开发，也支持分支开发。
  4. 自定义工作流：自定义工作流是指根据实际情况，结合集中式、功能分支工作流和发布工作流的特点，制定一套适合自己的工作流。

Git 基本命令 ​

Git 基本命令是 Git 最常用的命令。Git 基本命令有以下几种：

  1. git init：初始化一个 Git 仓库。
  2. git add：添加文件到暂存区。
  3. git commit：提交文件到本地仓库。
  4. git clone：克隆一个远程仓库到本地。
  5. git push：推送本地仓库的修改到远程仓库。
  6. git pull：从远程仓库拉取最新代码到本地。
  7. git branch：创建、切换、删除分支。
  8. git checkout：切换分支。
  9. git merge：合并分支。
  10. git log：查看提交历史。
  11. git status：查看仓库状态。

Git 开发使用 ​

Git 安装 ​

Git 可以在各个平台上安装，包括 Windows、Mac、Linux 等。Git 安装非常简单，只需要到 Git 官方网站下载安装包，然后按照提示一步步安装就可以了。

Git 配置 ​

Git 的配置包括全局配置和本地配置。全局配置是针对用户的，而本地配置是针对仓库的。

全局配置 ​

全局配置包括用户名和邮箱，以及 Git 的编辑器。

git config --global user.name "your name"
git config --global user.email "your email"
git config --global core.editor "your editor"

发布工作流 ​

发布工作流是将代码库存放在中央服务器，开发者克隆代码到本地，然后在本地进行开发，开发完成后将代码打包，然后发布到测试环境进行测试，测试通过后再发布到生产环境。发布工作流支持多人同时开发，也支持分支开发。

1. 开发分支：开发分支是指开发人员在本地创建的分支，用于开发新功能或修复 bug。

  1. 发布分支：发布分支是指测试人员在本地创建的分支，用于发布测试版本。

  2. 主分支：主分支是指代码库的主分支，是所有分支的最终汇总。

  3. 合并分支：合并分支操作会将指定分支的代码合并到当前工作分支。

  4. 打包发布：打包发布操作会将代码打包，并发布到测试环境进行测试。

  5. 发布测试：发布测试操作会将测试通过的代码发布到生产环境。

Git 常用命令 ​

Git 常用命令包括初始化、添加、提交、克隆、推送、拉取、分支切换、创建分支、切换分支、合并分支、删除分支、打包发布、发布测试、回滚版本等。

  1. 初始化仓库：git init
  2. 添加文件到暂存区：git add file-name
  3. 提交文件到本地仓库：git commit -m "commit message"
  4. 克隆远程仓库到本地：git clone https://github.com/username/repository.git
  5. 推送本地仓库的修改到远程仓库：git push
  6. 从远程仓库拉取最新代码到本地：git pull
  7. 创建分支：git branch branch-name
  8. 切换分支：git checkout branch-name
  9. 合并分支：git merge branch-name
  10. 删除分支：git branch -d branch-name
  11. 打包发布：git checkout release && git merge develop && git tag v1.0.0 && git push origin release --tags
  12. 发布测试：git checkout master && git merge release && git push origin master
  13. 回滚版本：git reset --hard v1.0.0
